# 0044 doria-rt internal ABI

Status: Accepted

Forward note: decision 0045 extends this private ABI in Stage 16 with runtime strings and display conversion. Decision 0074 extends it in Stage 17 with raw standard-device I/O, explicit flush, per-stream interactivity detection, buffered line input, UTF-8 text-file operations, nullable-string equality, and validated-format-plan helpers. All remain compiler-internal and versioned with compiler/runtime lockstep.

## Decision

`doria-rt` is the Rust bootstrap runtime static library for native Doria programs. It is compiler-internal and unstable before 1.0; compiler and runtime are versioned in lockstep.

Runtime ABI symbols are `dr_`-prefixed. Stage 12 uses versioned `dr_v1_*` symbols so incompatible compiler/runtime combinations fail clearly at link time. The initial ABI provides contracts materially equivalent to:

```text
dr_v1_main_int(entry_function_pointer) -> i32
dr_v1_main_void(entry_function_pointer) -> i32
dr_v1_write_stdout(current_frame, bytes_pointer, byte_length)
dr_v1_write_stderr(bytes_pointer, byte_length)
dr_v1_panic(current_frame, message_pointer, message_length) -> never
```

Generated Doria functions maintain a C-layout `DrStackFrameV1` containing a parent-frame pointer, source function-name byte pointer, and function-name byte length. This metadata is compiler-generated and is not source-visible MIR or a public ABI promise.

Stage 12 `doria-rt` owns:

- process entry policy
- exact stdout and stderr writes
- panic formatting and status 101
- Doria stack-frame traversal

The OS-required exported process symbol `main` is generated by the compiler and is not a public Doria symbol.

## Stage 12 constraints

The runtime performs no heap allocation and introduces no runtime string representation. Future allocator, string, collection, ownership, and I/O work extends this crate later. The current runtime uses exact platform writes and abort-only process termination.

## Stage 17 extension

The Stage 17 compiler may call private `dr_v1_*` operations for stdin line reads, stdout/stderr flush, independent standard-stream interactivity checks, UTF-8 text-file read/write, exact stderr string output, nullable-string equality, and the checked string/integer/float format conversions described by Decision 0074. These symbols encode validated MIR operations; they are not a stable C API or a public Doria stream API. Raw device access remains below line discipline, and Windows console-wide handling remains an implementation detail. The future Stage 46 `Console` surface builds on these primitives without exposing them directly.
